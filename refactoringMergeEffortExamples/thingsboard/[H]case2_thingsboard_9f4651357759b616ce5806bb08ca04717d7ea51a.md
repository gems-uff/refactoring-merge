P1 used the hardcoded string `"ALARM"` as a parameter to `TbMsg.newMsg(...)`. P2 replaced this with the type-safe enum constant `TbMsgType.ALARM` — a **Replace_Attribute** (replacing a magic string constant with an enum value). The `--`/`++` lines at the call site in `AlarmState` represent the merge effort to apply P2's type-safe version over P1's string literal.

### 2. Extract_Method — `isSendEventToAlarmRuleStateServiceAvailable`

P1 used a boolean variable `sendEventToAlarmRuleStateService` inline in the condition. P2 extracted this logic into a dedicated private method `isSendEventToAlarmRuleStateServiceAvailable(TbMsgType)` with an explicit switch expression covering all applicable message types. Note: the `isSendEventToAlarmRuleStateServiceAvailable` method itself is introduced as `++` lines — however, this is the merge applying the **Extract_Method** refactoring from P2 into the final merged code, not a refactoring invented by the merge itself. The method existed in P2; the `++` lines are the result of P1 not having it, creating a conflict that the merge resolves.
