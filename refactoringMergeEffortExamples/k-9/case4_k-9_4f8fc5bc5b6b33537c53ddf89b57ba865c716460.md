public static Body createBody(InputStream in, String contentTransferEncoding, String contentType)
        throws IOException, MessagingException {
    if (contentTransferEncoding != null) {
        contentTransferEncoding = MimeUtility.getHeaderParameter(contentTransferEncoding, null);
    }
    BinaryTempFileBody tempBody;
    if (MimeUtil.isMessage(contentType)) {
        tempBody = new BinaryTempFileMessageBody(contentTransferEncoding);
    } else {
        tempBody = new BinaryTempFileBody(contentTransferEncoding);
    }
    OutputStream out = tempBody.getOutputStream();
    try {
        IOUtils.copy(in, out);
    } finally {
        out.close();
    }
    return tempBody;
}

## What each side had

**Parent 1 (P1)** had `MimeUtility.decodeBody(InputStream, String contentTransferEncoding, String contentType)` as the factory method that decodes a raw byte stream into a `Body` object. `Message.getHeaderNames()` and `MimeMessage.setHeader()` declared `throws UnavailableStorageException`.

**Parent 2 (P2)** renamed the factory method from `decodeBody` to `createBody` to better reflect its purpose (it does not decode, it stores raw data for later decoding). P2 also widened the exception types on `setHeader()` and `getHeaderNames()` from the narrow `UnavailableStorageException` (a `MessagingException` subclass specific to storage) to the broader `MessagingException`, removing an unnecessary coupling to the storage layer.

## Interpretation

**Rename_Method** (`decodeBody` → `createBody` in `MimeUtility`): The name `decodeBody` was misleading — the old implementation actually decoded content-transfer-encoding inline (stripping base64/QP encoding from the stream). The new implementation stores the raw encoded data as-is in a `BinaryTempFileBody` and defers decoding to a separate new `decodeBody(Body)` overload. To avoid confusion with the new `decodeBody(Body body)` method (which actually decodes), the raw-stream factory was renamed to `createBody`. The `--` line confirms P2 had `public static Body decodeBody(InputStream in, ...)`. The `++` line shows the rename to `createBody`.

**Change_Parameter_Type** (`UnavailableStorageException` → `MessagingException` in `setHeader`/`getHeaderNames`): P2 narrowed the declared exception type from the storage-specific `UnavailableStorageException` to the more general `MessagingException`. Since `UnavailableStorageException extends MessagingException`, this is a widening of the exception type in the signature, decoupling the interface from the storage implementation. The change propagates through the abstract `Message.getHeaderNames()`, `MimeMessage.setHeader()`, and `ImapStore`'s inner message class.
